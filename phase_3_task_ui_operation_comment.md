1. 任务状态 与 人(我本人)状态的管理
  - 任务状态：
    - 一共有四种。针对每种，需要提供操作按钮
        - 新建。新增的任务处于新建状态，此时要有 开始 Drop 按钮
        - 进行中。新建任务点击开始按钮后，进入此状态。  此时要有  Done  Drop 按钮
        - 完成。进行中的按钮点击Done，进入此状态。  此时要提供  Redo  按钮
        - 废弃。其他状态点击Drop按钮时，要弹框要求填入废弃原因。填入并点击确定后，进入此状态。
    - 对于完成和废弃任务，要有一个地方能看到它们的列表。我建议在任务列表上方的bar里加入 完成 和 废弃任务的筛选


  - 人的状态。 我同一时刻只能干一件事，所以我的状态：  干某一个任务 切换到另一个任务 或者 AFK(away from keyboard)
    - 需要增加一张表，保存我的工作状态记录
    - 当我 take over 某个任务时，新增一条记录
      - 当我 afk 时，结束上一条记录
      - 当我take over其他任务时，结束上一条记录，新增一条新纪录
    - 工作状态记录要关联某个任务
    - 工作状态记录要记录
      - 开始时间：即take over时间
      - 结束时间：即afk或者take over其他任务的时间
    - 基于这种设计，每天就可以统计出我在某个任务上耗费了多少时间、AFK了多少时间。
    - 在任务界面最上方的bar的最右侧，加入 take over  和 AFK  按钮
    - 在 report中 现有的元素下方增加一个状态记录区。状态记录区的元素
      - 最上方要有一个tab bar
        - 左边是三个可切换的tab，分别是 天 周 月
        - 可切换tab右侧是一个时间筛选框，根据 天 周 月 有不同的展示
          - 天：  三个按钮  左箭头 (日期+今天按钮) 右箭头。   
            - 日期中默认显示当天的日期
            - 点击左箭头，日期显示-1
            - 点击右箭头，日期显示+1
            - 点击今天，切换回今天
          - 周：  三个按钮  左箭头 (日期+本周按钮) 右箭头
            - 日期中默认显示 当天所在的周的周一的日期
            - 点击左箭头， 日期显示-7天
            - 点击右箭头， 日期显示+7天
            - 点击本周，切换回 当天所在的周的周一的日期
          - 月：  三个按钮  左箭头 (年月+本月按钮) 右箭头
            - 日期中默认显示 当天所在的月，如：2026/4
            - 点击左箭头， 年月显示-1月
            - 点击右箭头， 日期显示+1月
            - 点击本月，切换回 当天所在的月
      - 下方是工作记录展示区，分左右两部分
        - 左：展示一个概览，它显示了上方tab bar中选中的时间范围（某天 or 某周 or 某月）包括：
          - 在岗时长，计算方式：
            - 每天的在岗时长为当天首个take over到当天最后一个AFK
            - 每周或每月的在岗时长为该周/月所有有工作记录的天数里，每天的在岗时长之和
          - 工作时长，计算方式：
            - 每天在岗时长中，所有工作记录的工作时长（每条工作记录的开始时间和结束时间）
            - 每周或每月的工作时长为该周/月所有有工作记录的天数里，每天的工作时长之和
          - 摸鱼时长，计算方式：
            - 每天在岗时长中 - 当天工作时长
            - 每周或每月的工作时长为该周/月所有有工作记录的天数里，每天的摸鱼时长之和
        - 右：展示一个列表，列表中是上方tab bar中已选的时间范围内的所有工作记录
          - 按时间倒序排列
          - 如果选择范围是周或者月，那么展示上要按天来group工作记录，同时在group最起始加上日期
          - 展示内容包括：开始时间 结束时间(上下排列，便于对比)，工作时长，关联的task

2. 交互和快捷键
  - 交互
    - 在任务管理按cmd + n
      - 进入新建任务功能
      - 如果当时正在take over其他任务，则结束当前的工作记录
    - 新建任务界面
      - 如果还没有输入任何内容
        - 如果点击了右上的cancel按钮，则
            - 退出编辑，不保存任何任务
            - 如果先前有在take over的任务，则切换回去先前的任务。并且开启一条新的关联先前任务的工作记录
        - 快捷键：esc或者cmd + w，等同于点了右上角的cancel
      - 如果输入了内容点右上角的cancel，则
        - 弹框提示是否确认取消。提供 确认和dismiss按钮。如果确认，
            - 退出编辑，不保存任何任务
            - 如果先前有在take over的任务，则切换回去先前的任务。并且开启一条新的关联先前任务的工作记录
        - 如果dismiss,则继续编辑
        - 快捷键：
          - 如果按了cmd + w，等同于点了右上角的cancel
          - 如果按了cmd + s或者esc，等同于点了提交
    - 查看任务界面
      - 新增工作记录的编辑框，
        - 如果当前没有take over这个任务，则置灰。用鼠标点击一次编辑框后
          - 进入编辑状态
          - 如果当前已经在take over其他任务的话，自动开始take over这个任务
          - 如果此时点击了esc 
            - 如果输入了内容，则等同于点击了提交
            - 如果还没有输入内容，则等同于点了cancel
          - 此时如果点击了cmd + w，则等同于点了cancel
      - 点击某条工作记录进入编辑态时：
        - 隐藏最下方的新增工作记录编辑框
        - 如果此时点击了esc 
          - 如果输入了内容，则等同于点击了提交
          - 如果还没有输入内容，则等同于点了cancel
        - 此时如果点击了cmd + w，则等同于点了cancel
