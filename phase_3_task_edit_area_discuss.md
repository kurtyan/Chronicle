1. 任务的数据结构相关调整：
  - 标题
  - 去掉任务描述。按以下思路修改：
    我在操作任务的时候，最初创建任务会输入一个富文本作为任务初始的描述（也可以叫做任务正文）。后面在操作任务时，可以追加任务记录。
    我建议的数据结构：  
    - 把任务正文与任务记录同等对待
    - 使用一个list保存所有任务正文与任务记录

2. 创建任务界面的调整
  - 取消现在的弹窗新增方式
  - 改成： 当点击创建任务时
    - 在当前的任务列表中的最上方新增一个entry
    - 右侧任务编辑区展示一个空的任务
      - 标题处于可输入状态，在输入进度中，任务列表中的entry实时同步展示已输入的标题。点击回车时，不要做数据存储，而是光标跳转到下方的任务正文编辑框
      - 下方展示任务正文编辑框
      - 点击ctrl + s时，完成任务创建。


3. 任务展示界面的调整
  - 任务正文和所有任务记录 以 富文本块的方式展示，同时展示一个新的空的富文本编辑框用来快速录入新进展
  - 当点击任意一个已有任务正文或者任务记录时，把该条的“富文本块“切换成“富文本编辑器“，用来编辑任务正文或者记录。编辑后，不改变任务正文/记录的创建时间，因此也不要改变该条目在list中的展示顺序

4. 按照#3中的逻辑，任务不存在显式的 编辑/展示 两种状态，而是只有一种状态： 展示、且随时可以编辑
